## 1. Arquitetura Geral

```
Lead abre o chat → Responde fase + gargalo → Informa URL do site
       ↓
Jina AI Reader raspa o site (bypassa Cloudflare/JS)
       ↓
Gemini 2.5 Flash cruza: site + fase + gargalo + serviços Connect Ideia
       ↓
Diagnóstico em 3 partes aparece no chat (negócio / gap / ação)
       ↓
Lead informa e-mail + WhatsApp → dados salvos no Supabase
```

## 3. Backend — `/api/analyze.js`

**Tecnologia:** Node.js Serverless (Vercel), ES Modules (`"type": "module"` no package.json)

**Fluxo:**
1. Recebe `{ url, phase, bottleneck }` do chat
2. Usa **Jina AI Reader** (`https://r.jina.ai/URL`) para scraping confiável
   - Bypassa Cloudflare, lida com JavaScript, devolve texto limpo
   - Timeout: 15 segundos
   - Se falhar → retorna `{ skipped: true }` e chat avança silenciosamente
3. Cruza conteúdo do site com respostas do chat via prompt do Gemini
4. Retorna diagnóstico estruturado em JSON:
```json
{
  "negocio": "O que o lead faz (identificação precisa)",
  "gap": "Principal problema digital detectado no site",
  "acao": "Como a Connect Ideia resolve com resultado tangível"
}
```

**Mapa de serviços Connect Ideia (hardcoded no backend):**
- LP / Landing Page → Landing Pages de Alta Conversão
- App / PWA → Apps customizados Discovery→GoLive
- Automação → WhatsApp + CRM + Fluxos inteligentes
- Ecossistema → LP + App + Automação integrados

## 4. Backend — `/api/save-lead.js`

**Tecnologia:** Node.js Serverless + `@supabase/supabase-js`

**Tabela Supabase:** `leads_chat`

**Schema da tabela (criar com SQL abaixo se não existir):**
```sql
create table if not exists leads_chat (
  id bigint generated by default as identity primary key,
  phase text,
  bottleneck text,
  site_url text,
  timing text,
  email text,
  whatsapp text,
  ai_summary text,
  created_at timestamptz default now()
);

alter table leads_chat enable row level security;
create policy "Allow public inserts" on leads_chat for insert with check (true);
```

**Campos salvos:**
- `phase` → Fase do projeto (Ideia/Operação/Escala)
- `bottleneck` → Gargalo declarado pelo lead
- `site_url` → URL informada para análise
- `timing` → Urgência do projeto
- `email` → E-mail corporativo
- `whatsapp` → WhatsApp
- `ai_summary` → Diagnóstico completo gerado pela IA (negocio | gap | acao)

## 5. Frontend — Chat Conversacional (`index.html`)

**Tecnologia:** HTML/JS vanilla injetado no final do body

**Fluxo do Chat (State Machine):**
1. **Fase** (options): Ideia / Operação / Escala
2. **Gargalo** (options): LP / App / Automação / Ecossistema
3. **Site** (text/url): URL → dispara análise cruzada Jina+Gemini → exibe 3 balões
4. **Timing** (options): Urgência do projeto
5. **E-mail** (text/email): Validação de formato
6. **WhatsApp** (text/tel): Validação de dígitos
7. **End**: Salva no Supabase e exibe mensagem final

**Proteções UX:**
- `isProcessing` flag → previne double-submit e fechamento acidental durante análise
- Validação de e-mail e WhatsApp com feedback visual (flash vermelho)
- Modal não fecha durante o processamento da IA

**Exibição da análise (3 balões com pausa dramática):**
```
🏢 [NEGÓCIO identificado]           ← 1.2s de pausa
⚠️ [GAP detectado no site]          ← 1.5s de pausa  
🚀 [AÇÃO = serviço Connect Ideia]   ← 2.0s de pausa
```

## 9. Decisões Técnicas Importantes

| Decisão | Motivo |
|---|---|
| Jina AI Reader em vez de fetch direto | Fetch direto é bloqueado por Cloudflare e sites com JS |
| Gemini 2.5 Flash | Único modelo disponível na API key atual sem erro 404 |
| `"type": "module"` no package.json | Vercel Node 18+ exige ES Modules para usar `import` |
| Diagnóstico em JSON estruturado | Permite formatar 3 balões separados no chat com emoção |
| Sem análise por domínio | Análise inventada é pior que silêncio — só analisa se leu o site |
